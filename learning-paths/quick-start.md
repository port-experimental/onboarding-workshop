# Quick Start Learning Path

*Perfect for experienced developers who want to get up and running quickly*

## Overview

**Duration**: 2-3 hours  
**Prerequisites**: Familiarity with APIs, YAML, and developer tools  
**Focus**: Essential Port concepts with hands-on examples

## Learning Objectives

By the end of this path, you will be able to:
- Navigate Port effectively and understand core concepts
- Create basic blueprints and configure data sources
- Build simple dashboards and self-service actions
- Understand Port's value proposition for your organization

## Your Learning Journey

### Phase 1: Core Concepts (45 minutes)
**Get the essentials quickly**

1. **[Getting Started](../modules/01-getting-started/)** (15 min)
   - Skip detailed exploration, focus on navigation
   - Understand the four main sections: Home, Catalog, Self-Service, Builder

2. **[Blueprints](../modules/02-blueprints/)** (15 min)
   - Focus on the GitHub Release blueprint exercise
   - Understand properties, relationships, and meta properties

3. **[Data Sources](../modules/03-data-sources/)** (15 min)
   - Complete the GitHub integration mapping
   - Understand JQ basics and data transformation

### Phase 2: Practical Application (60-90 minutes)
**Build something useful**

4. **[Dashboards](../modules/04-dashboards/)** (30 min)
   - Create one comprehensive dashboard
   - Focus on table and chart widgets
   - Skip advanced customization

5. **[Actions](../modules/05-actions/)** (45 min)
   - Create the deployment action exercise
   - Understand payload templating
   - Skip complex approval workflows

6. **[Scorecards](../modules/06-scorecards/)** (15 min)
   - Create one simple scorecard
   - Understand level logic
   - Skip advanced rule patterns

### Phase 3: Next Steps (30 minutes)
**Plan your implementation**

7. **[Terraform](../modules/07-terraform/)** (30 min)
   - Understand infrastructure-as-code benefits
   - Set up basic provider configuration
   - Skip complex CI/CD setup

## Key Takeaways for Quick Start

- **Port is a Developer Portal**: Centralized place for all development resources
- **Blueprints Define Structure**: Think database schema for your software catalog
- **Integrations Populate Data**: Automatic data ingestion from GitHub, CI/CD, etc.
- **Self-Service Reduces Friction**: Developers can do more without waiting
- **Quality is Measurable**: Scorecards provide objective quality metrics

## Implementation Checklist

After this quick start, you should be able to:

- [ ] Explain Port's value to stakeholders
- [ ] Design basic blueprints for your organization
- [ ] Configure GitHub integration for your repositories
- [ ] Create dashboards for team visibility
- [ ] Build simple self-service actions
- [ ] Plan a Port rollout strategy
