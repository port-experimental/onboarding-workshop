# Developer Learning Path

*Perfect for developers who will use Port for day-to-day development tasks*

## Overview

**Duration**: 4-6 hours  
**Prerequisites**: Basic development experience  
**Focus**: Using Port for self-service, dashboards, and development workflows

## Learning Objectives

By the end of this path, you will be able to:
- Navigate Port effectively for daily development tasks
- Use self-service actions to automate common workflows
- Create and customize dashboards for your team
- Understand how Port integrates with your development tools
- Contribute to your organization's developer portal

## Your Learning Journey

### Phase 1: Foundation (1-2 hours)
**Get comfortable with Port basics**

1. **[Getting Started](../modules/01-getting-started/)** (30 min)
   - Port navigation and interface
   - Understanding the software catalog
   - Your first Port interaction

2. **[Blueprints](../modules/02-blueprints/)** (45 min)
   - What are blueprints and entities?
   - How your services and teams are modeled
   - Finding and understanding your data

3. **[Data Sources](../modules/03-data-sources/)** (45 min)
   - How Port connects to GitHub and other tools
   - Understanding data flow and updates
   - Troubleshooting data issues

### Phase 2: Daily Usage (2-3 hours)
**Learn the tools you'll use every day**

4. **[Dashboards](../modules/04-dashboards/)** (60 min)
   - Creating personal and team dashboards
   - Understanding widgets and visualizations
   - Customizing views for your workflow

5. **[Actions](../modules/05-actions/)** (90 min)
   - Using self-service actions
   - Common developer workflows
   - Creating simple actions for your team

### Phase 3: Advanced Usage (1 hour)
**Become a power user**

6. **[Scorecards](../modules/06-scorecards/)** (60 min)
   - Understanding quality metrics
   - Using scorecards to improve your services
   - Contributing to quality standards

## Key Takeaways for Developers

- **Self-Service First**: Port empowers you to do more without waiting for others
- **Visibility Matters**: Good dashboards help you and your team stay informed
- **Automation Saves Time**: Actions can eliminate repetitive manual tasks
- **Quality is Measurable**: Scorecards help you maintain and improve service quality
- **Collaboration**: Port makes it easier to work across teams and services

## Quick Reference

### Daily Developer Tasks in Port
- Check service health and metrics
- Run self-service actions (deployments, environment setup, etc.)
- View team dashboards and reports
- Find service documentation and contacts
- Monitor quality scores and improvements

### Common Port URLs (Bookmarks)
- Software Catalog: Your main dashboard
- Self-Service: Available actions
- Your Team's Dashboard: Custom team view
- Service Documentation: Centralized docs
